4 pivotハンドリング - RとSQLでの比較

ここでは、long-wide変換であるpivotを取り上げます。pivot(long-wide)変換自体の説明については省略します。

4.1 データの準備

いわゆるlong型になっているデータとしては、dbにはt_df_logがlong型になってます。なのでこれを今回は利用します。

wide型になっているデータとしては、dbには書き込みテストで利用した t_irisがあります。 これを利用してみましょう。

4.2 from long to wide

long型からwide型へ変換します。t_df-logで、item別個数を横へ広げます。レコードはユーザー単位です。つまり、｢ユーザーごとに各アイテムを何個ずつ購入したか｣を示す表になります。

4.2.1 expand to wide by SQL

先にSQLで記述します。

別ファイルでsqlを書いて、それを呼び出すこととします。

# library
library(tidyverse)
library(DBI)


# 先にsqlファイルを読み込む関数を準備
read_sql <- function(file) {
  string <- readLines(file)
  res <- paste(string, collapse = "\n")
  return(res)
}

# queryを格納
query_02_wider <- read_sql("sql/pivot_wide.sql")
# queryを表示
cat(query_02_wider)
## select
##   id
##   ,sum(case when item = 'item_1' then 1 end) as item_1
##   ,sum(case when item = 'item_2' then 1 end) as item_2
##   ,sum(case when item = 'item_3' then 1 end) as item_3
##   ,sum(case when item = 'item_4' then 1 end) as item_4
##   ,sum(case when item = 'item_5' then 1 end) as item_5
## from
##   t_df_log
## group by
##   id
# 問い合わせる
con <- dbConnect(con)
df_res_02_sql <- dbGetQuery(con,
                            query_02_wider)
head(df_res_02_sql)
##        id item_1 item_2 item_3 item_4 item_5
## 1 1000001     19     12      2     NA     NA
## 2 1000002     20     14      1      2     NA
## 3 1000003     25      6      1      1     NA
## 4 1000004     23     13      3     NA     NA
## 5 1000005     20      8      5      1     NA
## 6 1000006     19     22      2      1     NA

SQLiteでwideに展開しようとするなら、基本的には上記のようなクエリになります。unitで考えると以下のようになります:

  1. unit_1: pivot_wide
    1. 入力: t_df_logを準備
    2. 削除:
      1. 列選択: idとitemだけを取り出す
      2. (行選択はスキップ)
    3. 加工:
      1. item -> item_1
        1. 変換: item = item_1なら1、そうでないなら0
        2. 集約: これをsum
      2. item -> item_2
        1. 変換: item = item_2なら1、そうでないなら0
        2. 合計: これをsum
      3. item -> item_3
        1. 変換: item = item_3なら1、そうでないなら0
        2. 合計: これをsum
      4. item -> item_4
        1. 変換: item = item_4なら1、そうでないなら0
        2. 合計: これをsum
      5. item -> item_5
        1. 変換: item = item_5なら1、そうでないなら0
        2. 合計: これをsum
    4. 集約:
      1. 集約化変数: id
    5. (結合はスキップ)

今回のsqlではunit_1だけになりましたが、他の書き方もあります:

# queryを格納
query_02_wider2 <- read_sql("sql/pivot_wide2.sql")
# queryを表示
cat(query_02_wider2)
## select
##   id
##   ,max(case when item = 'item_1' then cnt end) as item_1
##   ,max(case when item = 'item_2' then cnt end) as item_2
##   ,max(case when item = 'item_3' then cnt end) as item_3
##   ,max(case when item = 'item_4' then cnt end) as item_4
##   ,max(case when item = 'item_5' then cnt end) as item_5
## from (
##   select
##     id
##     ,item
##     ,count(*) as cnt 
##   from
##     t_df_log
##   group by
##     id
##     ,item
## ) unit_1
## group by
##   id
# 問い合わせる
con <- dbConnect(con)
df_res_02_sql2 <- dbGetQuery(con,
                             query_02_wider2)
head(df_res_02_sql2)
##        id item_1 item_2 item_3 item_4 item_5
## 1 1000001     19     12      2     NA     NA
## 2 1000002     20     14      1      2     NA
## 3 1000003     25      6      1      1     NA
## 4 1000004     23     13      3     NA     NA
## 5 1000005     20      8      5      1     NA
## 6 1000006     19     22      2      1     NA

今回はこのような処理になります:

  1. unit 1: id, itemでcount集計
    1. 入力: t_df_log
    2. 削除: id, itemを選択
    3. 加工:
      1. count: -> cnt
    4. 集約:
      1. 集約化変数: id, item
    5. (結合はないのでスキップ)
    6. 出力: unit_1
  2. unit 2: pivot wide
    1. 入力: unit_1
    2. 削除:
      1. 列選択: idを取り出す
    3. 加工:
      1. item -> item_1
        1. 変換: item = item_1ならcnt、そうでないならnull
        2. 集約: これをmax
      2. item -> item_2
        1. 変換: item = item_2ならcnt、そうでないならnull
        2. 集約: これをmax
      3. item -> item_3
        1. 変換: item = item_3ならcnt、そうでないならnull
        2. 集約: これをmax
      4. item -> item_4
        1. 変換: item = item_4ならcnt、そうでないならnull
        2. 集約: これをmax
      5. item -> item_5
        1. 変換: item = item_5ならcnt、そうでないならnull
        2. 集約: これをmax
    4. 集約:
      1. 集約化変数: id
    5. (結合はスキップ)

unit 2の加工でmaxを使っているけど、これは対象レコードが1しかないはずなので、 別にmaxでもminでもどれでも同値になります(でもなぜかmaxを利用する例が多い)。

最初の例は、この2つめの例をショートカットしたものとなります。集約ロジックがただのcountなのであんまり変わらないですが、複雑な計算をさせる場合は効率性などは意識したほうがいいでしょう。
